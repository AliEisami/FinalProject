# QA Automation for Trello Website (Final Project - Part 1)

## Project Overview
QA Automation for Trello Website aims to automate the testing of Trello's core functionalities using tools like
Selenium WebDriver and Python. The scope includes automating user authentication,
board creation and management, task creation and management, as well as profile management.

## Purpose of the Project
QA Automation for Trello Website aims to automate the testing of Trello's core functionalities using tools like Selenium
 WebDriver and Python. The scope includes automating user authentication, board creation and management,
  task creation and management, as well as profile management.

## Key Features
- **Functional Testing**: Test cases will be created for each feature and functionality of the Trello website.
- **Usability Testing**: User scenarios will be tested to ensure the website is easy to use.
- **Compatibility Testing**: The website will be tested on different browsers and devices.

## Technologies and Tools Used
- **Programming Language**: Python
- **Automation Tool**: Selenium WebDriver

## Installation Instructions
To set up the project locally, follow these steps:

1. **Clone the repository**:
    ```sh
    git clone https://github.com/AliEisami/Final_Project.git
    ```

2. **Navigate to the project directory**:
    ```sh
    cd Final_Project
    ```

3. **Set up a virtual environment (optional but recommended)**:
    ```sh
    python -m venv venv
    source venv/bin/activate  # On Windows use `venv\Scripts\activate`
    ```

4. **Install the required dependencies**:
    ```sh
    pip install -r requirements.txt
    ```

## Usage Instructions
To run the automated tests, use the following commands:

1. **Activate the virtual environment (if you set one up)**:
    ```sh
    source venv/bin/activate  # On Windows use `venv\Scripts\activate`
    ```

2. **Run the tests**:
    ```sh
    pytest
    ```

3. **Generate a test report**:
    ```sh
    pytest --html=report.html
    ```
